# routes-analysis 需求设计
## 🔖 项目定位
华为交换机BGP路由表多场景分析工具，用于运维路由巡检、配置核查、负载分担合理性检查。
> 原`routescompare`项目已废弃，全部功能整合至此项目。
## 🎯 核心功能
| 场景 | 子命令 | 核心价值 |
|------|--------|----------|
| 多组平行设备负载分担分析 | `multi-group` | 检查路由是否分担到多组不同的平行设备，发现单归、负载不均问题 |
| 多设备BGP路由表比较 | `compare` | 多台设备路由表差异对比，发现配置遗漏、路由缺失问题 |
## 📁 目录架构
```

├── tests/                   # 单元测试+测试用例
├── scripts/                 # 打包脚本

```
## 🧩 数据流程
### multi-group 场景
```
输入文件 → Parser解析 → 生成Device对象（包含路由表+接口对端映射）
→ Analyzer分析 → 分组对端设备，统计每个目的网段的所属组数量
→ 筛选出≥min_groups的命中路由 → Exporter生成Excel报告
```
### compare 场景
```
多个输入文件 → Parser解析 → 生成多个Device对象
→ Comparator设置基准设备 → 逐对比较所有设备和基准的路由差异
→ 统计4类差异（缺失网段/缺失接口/接口不匹配/优先级开销差异）
→ Exporter生成带颜色标记的Excel差异报告
```
## 🔢 版本管理
采用简化语义化版本：`主版本号.次版本号.修订号`
- 主版本号：架构大改/不兼容变更
- 次版本号：新增功能/兼容变更
- 修订号：Bug修复/体验优化
发布流程：修改`pyproject.toml`版本号 → `git tag vx.y.z` → 推送远端触发自动打包
