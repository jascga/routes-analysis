# 优化规划
优先级从高到低排序：
---
## 🔝 第一优先级：解决业务痛点，改完马上能用
### 1. 输入层抽象，支持多厂商/多数据源
- 支持SSH直接读取交换机输出、网管平台拉取、zip压缩包直接解析
- 抽象`BaseParser`基类，支持华为/华三/思科/Juniper多厂商路由表自动识别解析
- 增强预处理能力：支持更多编码识别、自动去控制字符、合并多行ECMP路由
### 2. 规则配置化，不用改代码调整逻辑
- 支持`config.yaml`配置驱动
- 可配置平行设备分组规则、差异告警阈值、导出字段、忽略网段/接口规则
---
## 🚀 第二优先级：提升扩展性/性能，适配大场景
### 3. 分析层插件化，新增场景零侵入
- 每个分析场景是`plugins/`目录下的独立插件，实现统一接口
- 程序启动自动扫描插件，自动生成CLI子命令
- 新增场景不用修改核心代码
### 4. 统一输出层，支持多格式导出
- 抽象`BaseExporter`基类，统一API
- 支持多格式：Excel/CSV/HTML/JSON/Markdown
- 支持多输出目的地：本地文件/邮件告警/企业微信/对象存储
### 5. 缓存+并行分析，提升大文件/批量场景速度
- 解析结果本地缓存，重复分析大文件速度提升10倍+
- 多文件解析/分析多进程并行，100台设备分析从分钟级降到秒级
---
## 📈 第三优先级：体验/生态增强
### 6. 数据模型抽象，支持全协议路由
- 抽象`BaseRoute`基类，公共字段提基类，各协议特有字段放子类
- 支持静态/OSPF/ISIS等全协议路由分析
### 7. 轻量Web UI
- FastAPI实现简单页面，支持上传路由表、在线分析、下载报告
- 可部署为内部巡检服务
### 8. 内置规则引擎，支持自定义告警
- YAML配置自定义检查规则，自动输出告警报告
- 覆盖90%自定义分析需求，不用改代码
